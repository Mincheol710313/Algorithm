## 📝 문제 설명

```text
회전 초밥 음식점에는 회전하는 벨트 위에 여러 가지 종류의 초밥이 접시에 담겨 놓여 있고, 손님은 이 중에서 자기가 좋아하는 초밥을 골라서 먹는다. 초밥의 종류를 번호로 표현할 때, 다음 그림은 회전 초밥 음식점의 벨트 상태의 예를 보여주고 있다. 벨트 위에는 같은 종류의 초밥이 둘 이상 있을 수 있다.

새로 문을 연 회전 초밥 음식점이 불경기로 영업이 어려워서, 다음과 같이 두 가지 행사를 통해서 매상을 올리고자 한다.

1. 원래 회전 초밥은 손님이 마음대로 초밥을 고르고, 먹은 초밥만큼 식대를 계산하지만, 벨트의 임의의 한 위치부터 k개의 접시를 연속해서 먹을 경우 할인된 정액 가격으로 제공한다.
2. 각 고객에게 초밥의 종류 하나가 쓰인 쿠폰을 발행하고, 1번 행사에 참가할 경우 이 쿠폰에 적혀진 종류의 초밥 하나를 추가로 무료로 제공한다. 만약 이 번호에 적혀진 초밥이 현재 벨트 위에 없을 경우, 요리사가 새로 만들어 손님에게 제공한다.

위 할인 행사에 참여하여 가능한 한 다양한 종류의 초밥을 먹으려고 한다.
회전 초밥 음식점의 벨트 상태, 메뉴에 있는 초밥의 가짓수, 연속해서 먹는 접시의 개수, 쿠폰 번호가 주어졌을 때, 손님이 먹을 수 있는 초밥 가짓수의 최댓값을 구하는 프로그램을 작성하시오.
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

슬라이딩 윈도우로 풀어야 한다는 건 알았는데, 벨트가 **원형**으로 연결되어 있어서 일반 선형 슬라이딩 윈도우를 그대로 쓸 수 없다는 점이 걸렸다.

### 2. 핵심 아이디어

**원형 배열 → 선형 배열로 변환**: 배열을 뒤에 한 번 더 이어 붙이면 원형 탐색을 선형으로 처리할 수 있다.

```
원래:  [7, 9, 7, 30, 2, 7, 9, 25]
복사:  [7, 9, 7, 30, 2, 7, 9, 25, 7, 9, 7, 30, 2, 7, 9, 25]
```

윈도우 시작점 `i`는 `0 ~ N-1`까지, 크기는 K로 고정. 배열 크기는 최소 `N+K-1`이 필요하므로 `2*N`으로 잡으면 항상 안전하다.

**윈도우 이동 로직** (K=4, C=30, 예시 일부):

| i | 윈도우 | distinct | count[30] | result |
|---|--------|----------|-----------|--------|
| 초기(0~3) | [7, 9, 7, 30] | 3 | 1 | 3 |
| 4 | [9, 7, 30, 2] | 4 | 1 | 4 |
| 5 | [7, 30, 2, 7] | 3 | 1 | 4 |
| 6 | [30, 2, 7, 9] | 4 | 1 | 4 |
| 7 | [2, 7, 9, 25] | 4 | 0 | **5** (쿠폰 +1) |

**쿠폰 처리**: 현재 윈도우에 쿠폰 번호 C가 없으면 +1 추가.

```cpp
result = max(result, distinct + (susi_count[C] == 0 ? 1 : 0));
```

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)` — 배열을 한 번 순회
- 공간 복잡도: `O(N + D)` — 복사 배열 + 카운트 배열

-----

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int N, D, K, C;
vector<int> susi_arr;
vector<int> susi_count;

int max_distinct() {
    int distinct = 0;

    for (int i = 0; i < K; i++) {
        if (susi_count[susi_arr[i]] == 0) distinct++;
        susi_count[susi_arr[i]]++;
    }

    int result = distinct + (susi_count[C] == 0 ? 1 : 0);

    for (int i = K; i < N+K-1; i++) {
        if(susi_count[susi_arr[i]] == 0) distinct++;
        susi_count[susi_arr[i]]++;

        susi_count[susi_arr[i-K]]--;
        if(susi_count[susi_arr[i-K]] == 0) distinct--;

        result = max(result, distinct + (susi_count[C] == 0 ? 1 : 0));
    }

    return result;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cin >> N >> D >> K >> C;
    susi_arr.assign(2*N, 0);
    susi_count.assign(D+1, 0);

    for (int i = 0; i < N; i++) {
        cin >> susi_arr[i];
        susi_arr[N+i] = susi_arr[i];
    }

    cout << max_distinct();

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|케이스 1 오답|초기 윈도우 중복 카운트 미반영 + 초기 result 쿠폰 미적용|

### 상세 원인

**버그 1: 초기 윈도우에서 count를 조건부로만 증가**

```cpp
// 틀린 코드
for (int i = 0; i < K; i++) {
    if (susi_count[susi_arr[i]] == 0) {
        distinct++;
        susi_count[susi_arr[i]]++;  // 중복 원소는 count가 1에 고정됨
    }
}

// 수정
for (int i = 0; i < K; i++) {
    if (susi_count[susi_arr[i]] == 0) distinct++;
    susi_count[susi_arr[i]]++;  // 항상 증가
}
```

초기 윈도우에 같은 초밥이 두 번 나오면 count가 1로 고정된다. 이후 슬라이딩할 때 그 초밥이 하나 빠지면 count가 0이 되어 distinct가 잘못 감소한다.

**버그 2: 초기 result에 쿠폰 미적용**

```cpp
// 틀린 코드
int result = distinct;

// 수정
int result = distinct + (susi_count[C] == 0 ? 1 : 0);
```

케이스 2가 우연히 맞은 이유는 초기 윈도우에 중복이 없고, C가 이미 윈도우 안에 있어서 두 버그가 모두 영향을 주지 않았기 때문이다.

### 개선점

- 슬라이딩 윈도우 초기화 구간도 본문 로직과 동일한 방식으로 작성할 것
- 초기 result 설정 시 쿠폰 조건 포함 여부를 반드시 확인할 것

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 원형 배열 → 2배 복사로 선형화

원형 구조에 슬라이딩 윈도우를 적용할 때, 배열을 뒤에 한 번 더 붙이면 선형 슬라이딩 윈도우로 처리할 수 있다. 배열 크기는 `N+K-1` 이상이면 충분하므로 `2*N`으로 잡으면 안전하다.

```cpp
for (int i = 0; i < N; i++) {
    susi_arr[N+i] = susi_arr[i];  // 뒤에 복사
}
// 윈도우 탐색: i = 0 ~ N-1, 루프 상한: N+K-1
```

### 2. 고정 크기 윈도우에서 distinct 원소 수 추적

`count[]` 배열로 윈도우 내 각 원소의 등장 횟수를 관리하면 O(1)으로 distinct 수를 유지할 수 있다.

```cpp
// 원소 추가 시
if (count[x] == 0) distinct++;
count[x]++;

// 원소 제거 시
count[x]--;
if (count[x] == 0) distinct--;
```
