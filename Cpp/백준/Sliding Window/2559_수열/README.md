## 📝 문제 설명

```text
매일 아침 9시에 학교에서 측정한 온도가 어떤 정수의 수열로 주어졌을 때, 연속적인 며칠 동안의 온도의 합이 가장 큰 값을 알아보고자 한다.

예를 들어, 아래와 같이 10일 간의 온도가 주어졌을 때,

3 -2 -4 -9 0 3 7 13 8 -3

모든 연속적인 이틀간의 온도의 합은 아래와 같다.
1 -6 -13 -9 3 10 20 21 5

매일 측정한 온도가 정수의 수열로 주어졌을 때, 연속적인 며칠 동안의 온도의 합이 가장 큰 값을 계산하는 프로그램을 작성하시오.
```

### 입력
```text
10 2
3 -2 -4 -9 0 3 7 13 8 -3
```

### 출력
```text
21
```

-----

## 🧠 풀이 과정

### 1. 처음 접근 방법

연속 K일의 합 중 최댓값을 구하는 문제이므로 고정 크기 슬라이딩 윈도우를 바로 떠올렸다.

### 2. 핵심 아이디어

크기 K의 윈도우를 오른쪽으로 한 칸씩 이동하면서 합을 갱신한다. 매번 새로 계산하지 않고, 새로 들어오는 원소를 더하고 빠지는 원소를 빼면 O(1)에 처리할 수 있다.

**윈도우 이동 예시** (N=10, K=2):

| i | 추가 | 제거 | sum | result |
|---|------|------|-----|--------|
| 초기(0~1) | - | - | 1 | 1 |
| 2 | -4 | 3 | -6 | 1 |
| 3 | -9 | -2 | -13 | 1 |
| 4 | 0 | -4 | -9 | 1 |
| 5 | 3 | -9 | 3 | 3 |
| 6 | 7 | 0 | 10 | 10 |
| 7 | 13 | 3 | 20 | 20 |
| 8 | 8 | 7 | 21 | **21** |
| 9 | -3 | 13 | 5 | 21 |

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)` — 배열을 한 번 순회
- 공간 복잡도: `O(N)` — 입력 배열

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int N, K;
vector<int> temp;

int max_temp() {
    int sum = 0;

    for (int i = 0; i < K; i++) sum += temp[i];

    int result = sum;

    for (int i = K; i < N; i++) {
        sum += temp[i];
        sum -= temp[i-K];
        result = max(result, sum);
    }

    return result;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cin >> N >> K;
    temp.assign(N, 0);

    for (int i = 0; i < N; i++) {
        cin >> temp[i];
    }

    cout << max_temp();

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|케이스 2 오답|루프 상한을 `N-K+1`로 잘못 설정해 윈도우가 중간에 멈춤|

### 상세 원인

```cpp
// 틀린 코드
for (int i = K; i < N-K+1; i++) { ... }

// 수정
for (int i = K; i < N; i++) { ... }
```

윈도우 시작점은 `0`부터 `N-K`까지 이동해야 하고, 루프에서 추가되는 원소의 인덱스는 `K`부터 `N-1`까지다. 따라서 상한은 `N`이어야 한다.

`N-K+1`로 하면 N=10, K=2일 때 `i < 9`가 되어 마지막 윈도우(`[8, -3]`)까지는 탐색하지만, K가 크면 루프 자체가 실행되지 않는다. 예를 들어 K=N이면 `i < 1`인데 루프는 `i=K=N`에서 시작하므로 한 번도 실행되지 않는다.

### 개선점

- 고정 크기 슬라이딩 윈도우에서 루프 상한은 항상 `N`으로 잡을 것
- 윈도우 시작 범위(`0 ~ N-K`)와 루프 변수 범위(`K ~ N-1`)를 혼동하지 말 것

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 고정 크기 슬라이딩 윈도우 기본 패턴

```cpp
// 초기 윈도우 합산
for (int i = 0; i < K; i++) sum += arr[i];
int result = sum;

// 윈도우 이동: i는 새로 들어오는 원소의 인덱스
for (int i = K; i < N; i++) {
    sum += arr[i];      // 오른쪽 원소 추가
    sum -= arr[i-K];    // 왼쪽 원소 제거
    result = max(result, sum);
}
```

루프 상한은 `N`으로 고정. `N-K+1` 같은 변형은 틀리기 쉽다.
