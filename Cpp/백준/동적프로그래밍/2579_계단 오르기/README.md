## 📝 문제 설명

```text
계단 오르기 게임은 계단 아래 시작점부터 계단 꼭대기에 위치한 도착점까지 가는 게임이다.
각각의 계단에는 일정한 점수가 쓰여 있는데 계단을 밟으면 그 계단에 쓰여 있는 점수를 얻게 된다.

계단 오르는 데는 다음과 같은 규칙이 있다.

1. 계단은 한 번에 한 계단씩 또는 두 계단씩 오를 수 있다.
2. 연속된 세 개의 계단을 모두 밟아서는 안 된다. 단, 시작점은 계단에 포함되지 않는다.
3. 마지막 도착 계단은 반드시 밟아야 한다.

각 계단에 쓰여 있는 점수가 주어질 때 이 게임에서 얻을 수 있는 총 점수의 최댓값을 구하는 프로그램을 작성하시오.
```

### 입력
```text
첫째 줄에 계단의 개수 N (1 ≤ N ≤ 300)
둘째 줄부터 N개의 줄에 각 계단의 점수 (1 이상 10,000 이하 자연수)

예시:
6
10
20
15
25
10
20
```

### 출력
```text
첫째 줄에 얻을 수 있는 총 점수의 최댓값을 출력한다.

예시:
75
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

처음에는 `dp[i]`를 "i번째 계단을 밟았을 때의 최대 점수"로 정의하고, 이전 계단에서 그냥 더해오려 했다.
하지만 `dp[i] = dp[i-1] + step[i]` 식으로 접근하면 연속 3칸 제약을 처리할 수 없었다.
`dp[i-1]`이 어디서 왔는지(연속 1칸인지 2칸인지) 알 수 없기 때문에, `i-2 → i-1 → i` 연속 3칸이 될 수 있다.

### 2. 핵심 아이디어

i번째 계단에 도달하는 경우는 정확히 두 가지뿐이다:

- **경우 1**: i-2 계단 → i 계단 (한 칸 건너뜀)
- **경우 2**: i-3 계단 → i-1 계단 → i 계단 (연속 2칸)

이 두 경우만 고려하면 연속 3칸 금지 조건을 자연스럽게 만족한다.

```
dp[i] = max(dp[i-2] + step[i], dp[i-3] + step[i-1] + step[i])
```

**dp 테이블 단계별 변화 예시** (입력: 10 20 15 25 10 20)

| i | step | 경우 1 (dp[i-2]+step[i]) | 경우 2 (dp[i-3]+step[i-1]+step[i]) | dp[i] |
|---|------|------------------------|--------------------------------------|-------|
| 1 | 10   | -                      | -                                    | 10    |
| 2 | 20   | -                      | -                                    | 30    |
| 3 | 15   | dp[1]+15 = 25          | dp[0]+step[2]+15 = 35                | 35    |
| 4 | 25   | dp[2]+25 = 55          | dp[1]+step[3]+25 = 50                | 55    |
| 5 | 10   | dp[3]+10 = 45          | dp[2]+step[4]+10 = 65                | 65    |
| 6 | 20   | dp[4]+20 = 75          | dp[3]+step[5]+20 = 65                | 75    |

최종 답: `dp[6] = 75` ✓

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)` — dp 배열을 한 번 순회
- 공간 복잡도: `O(N)` — dp 배열 및 step 배열 각 N+1 크기

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;

    vector<int> step_arr(N + 1, 0);
    for (int i = 1; i <= N; i++) {
        cin >> step_arr[i];
    }

    vector<int> dp(N + 1, 0);
    dp[1] = step_arr[1];
    dp[2] = step_arr[1] + step_arr[2];

    for (int i = 3; i <= N; i++) {
        dp[i] = max(dp[i-2] + step_arr[i], dp[i-3] + step_arr[i-1] + step_arr[i]);
    }

    cout << dp[N];

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차 (접근 시도)|잘못된 점화식|`dp[i-1] + step[i]` 사용 → 연속 3칸 가능성|

### 상세 원인

```cpp
// 잘못된 접근
dp[i] = dp[i-1] + step[i];  // dp[i-1]이 연속 2칸째일 수 있음 → 연속 3칸 위반
```

```cpp
// 올바른 접근
dp[i] = max(dp[i-2] + step_arr[i],                        // i-2에서 직접
            dp[i-3] + step_arr[i-1] + step_arr[i]);        // i-3 → i-1 → i
```

`dp[i-1]`의 값이 어느 경로로 왔는지 알 수 없기 때문에, `dp[i-1] + step[i]`는 쓸 수 없다.
대신 i번째 계단으로 올 수 있는 **합법적인 경로를 명시적으로 열거**해야 한다.

### 개선점

- DP 점화식을 세울 때 "이전 상태에서 더한다"는 접근보다 **"현재 상태에 도달하는 모든 경우를 열거"** 하는 방식으로 생각한다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. DP에서 경로 제약 조건 처리

제약이 있는 DP(연속 k개 금지 등)는 단순히 `dp[i-1]`에서 넘어오는 방식이 아니라, **현재 칸에 도달 가능한 합법적 경로를 명시적으로 나열**해야 한다.

```cpp
// 연속 3칸 금지 → 도달 가능한 두 경로만
dp[i] = max(
    dp[i-2] + step[i],               // 2칸 전에서 점프
    dp[i-3] + step[i-1] + step[i]   // 3칸 전 → 1칸 전 → 현재
);
```

### 2. 초기값 설정

N이 작을 때 `dp[i-3]`에서 음수 인덱스가 발생할 수 있으므로, `dp[0] = 0`으로 초기화하고 `dp[1]`, `dp[2]`는 직접 세팅한다.

```cpp
dp[1] = step[1];
dp[2] = step[1] + step[2];
// i = 3부터 점화식 적용
for (int i = 3; i <= N; i++) { ... }
```
