## 📝 문제 설명

```text
RGB거리에는 집이 N개 있다. 거리는 선분으로 나타낼 수 있고, 1번 집부터 N번 집이 순서대로 있다.

집은 빨강, 초록, 파랑 중 하나의 색으로 칠해야 한다. 각각의 집을 빨강, 초록, 파랑으로 칠하는 비용이 주어졌을 때, 아래 규칙을 만족하면서 모든 집을 칠하는 비용의 최솟값을 구해보자.

1번 집의 색은 2번 집의 색과 같지 않아야 한다.
N번 집의 색은 N-1번 집의 색과 같지 않아야 한다.
i(2 ≤ i ≤ N-1)번 집의 색은 i-1번, i+1번 집의 색과 같지 않아야 한다.
```

### 입력
```text
첫째 줄에 집의 수 N(2 ≤ N ≤ 1,000)이 주어진다. 둘째 줄부터 N개의 줄에는 각 집을 빨강, 초록, 파랑으로 칠하는 비용이 1번 집부터 한 줄에 하나씩 주어진다. 집을 칠하는 비용은 1,000보다 작거나 같은 자연수이다.

3
26 40 83
49 60 57
13 89 99
```

### 출력
```text
첫째 줄에 모든 집을 칠하는 비용의 최솟값을 출력한다.

96
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

처음에는 `dp[i] = dp[i-1] + min(이전 집과 다른 색을 칠하는 비용)` 형태로 단일 값으로 정의했다.
하지만 이렇게 하면 i번째 집이 어떤 색인지 알 수 없어서, 다음 집에 어떤 색이 금지인지를 판단할 수 없었다.

### 2. 핵심 아이디어

`dp[i]`를 **색깔별로 분리**하여 2차원 배열로 정의한다.

```
dp[i][0] = i번째 집을 R로 칠할 때 최소 비용
dp[i][1] = i번째 집을 G로 칠할 때 최소 비용
dp[i][2] = i번째 집을 B로 칠할 때 최소 비용
```

점화식:
```
dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + home[i][0]
dp[i][1] = min(dp[i-1][0], dp[i-1][2]) + home[i][1]
dp[i][2] = min(dp[i-1][0], dp[i-1][1]) + home[i][2]
```

**dp 테이블 단계별 변화 예시** (입력: 26 40 83 / 49 60 57 / 13 89 99)

| i | R 비용 | G 비용 | B 비용 | dp[i][0] (R) | dp[i][1] (G) | dp[i][2] (B) |
|---|-------|-------|-------|-------------|-------------|-------------|
| 0 | 26    | 40    | 83    | 26          | 40          | 83          |
| 1 | 49    | 60    | 57    | min(40,83)+49=89 | min(26,83)+60=86 | min(26,40)+57=83 |
| 2 | 13    | 89    | 99    | min(86,83)+13=96 | min(89,83)+89=178 | min(89,86)+99=188 |

최종 답: `min(96, 178, 188) = 96` ✓

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)` — dp 배열을 한 번 순회
- 공간 복잡도: `O(N)` — N×3 크기의 dp 배열

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;
    vector<vector<int>> home_arr(N, vector<int>(3, 0));

    for (int i = 0; i < N; i++) {
        for (int j = 0; j < 3; j++) {
            cin >> home_arr[i][j];
        }
    }

    vector<vector<int>> dp(N, vector<int>(3, 0));
    for (int i = 0; i < 3; i++) {
        dp[0][i] = home_arr[0][i];
    }

    for (int i = 1; i < N; i++) {
        dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + home_arr[i][0];
        dp[i][1] = min(dp[i-1][0], dp[i-1][2]) + home_arr[i][1];
        dp[i][2] = min(dp[i-1][0], dp[i-1][1]) + home_arr[i][2];
    }

    cout << min(dp[N-1][0], min(dp[N-1][1], dp[N-1][2]));

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|런타임 에러|`dp[i-2]` 사용 → i=1일 때 음수 인덱스 접근|
|2차|런타임 에러|`dp[N]` 사용 → 배열 범위 초과 (유효 인덱스는 N-1)|

### 상세 원인

**점화식 오류**: `dp[i-2]`를 사용하면 i=1일 때 `dp[-1]` 접근이 발생한다.

```cpp
// 잘못된 코드
dp[i][0] = min(dp[i-1][1], dp[i-2][2]) + home_arr[i][0];  // dp[i-2] 오류

// 올바른 코드
dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + home_arr[i][0];  // 이전 집(i-1)만 참조
```

**인덱스 오류**: dp 배열은 0-based로 크기 N이므로 마지막 인덱스는 N-1이다.

```cpp
// 잘못된 코드
cout << min(dp[N][0], min(dp[N][1], dp[N][2]));    // 범위 초과

// 올바른 코드
cout << min(dp[N-1][0], min(dp[N-1][1], dp[N-1][2]));
```

### 개선점

- 0-based 배열에서 마지막 원소는 항상 `[N-1]`임을 의식한다.
- 점화식에서 참조하는 인덱스가 음수가 되는 경우를 먼저 체크한다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 상태에 선택지가 있는 DP는 차원을 추가한다

단순히 `dp[i]` 하나로는 "어떤 선택을 했는지" 알 수 없다. 선택지가 K개라면 `dp[i][k]`로 차원을 늘려야 다음 상태에서 제약을 올바르게 적용할 수 있다.

```cpp
// 색깔 제약이 있는 경우 → 색깔별로 dp 분리
dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + cost[i][0];  // R
dp[i][1] = min(dp[i-1][0], dp[i-1][2]) + cost[i][1];  // G
dp[i][2] = min(dp[i-1][0], dp[i-1][1]) + cost[i][2];  // B
```

### 2. 3개 값의 최솟값

`min()`은 인자 두 개만 받으므로 중첩하거나 `initializer_list`를 활용한다.

```cpp
min(a, min(b, c))          // 중첩
min({a, b, c})             // initializer_list (C++11 이상)
```
