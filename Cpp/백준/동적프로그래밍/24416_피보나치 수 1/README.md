## 📝 문제 설명

```text
오늘도 서준이는 동적 프로그래밍 수업 조교를 하고 있다. 아빠가 수업한 내용을 학생들이 잘 이해했는지 문제를 통해서 확인해보자.

오늘은 n의 피보나치 수를 재귀호출과 동적 프로그래밍으로 구하는 알고리즘을 배웠다. 재귀호출에 비해 동적 프로그래밍이 얼마나 빠른지 확인해 보자. 아래 의사 코드를 이용하여 n의 피보나치 수를 구할 경우 코드1 코드2 실행 횟수를 출력하자.

피보나치 수 재귀호출 의사 코드는 다음과 같다.

fib(n) {
    if (n = 1 or n = 2)
    then return 1;  # 코드1
    else return (fib(n - 1) + fib(n - 2));
}

피보나치 수 동적 프로그래밍 의사 코드는 다음과 같다.

fibonacci(n) {
    f[1] <- f[2] <- 1;
    for i <- 3 to n
        f[i] <- f[i - 1] + f[i - 2];  # 코드2
    return f[n];
}
```

### 입력
```text
5
```

### 출력
```text
5 3
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

재귀 함수 `fib()` 안에 지역 변수 `count`를 선언해서 베이스 케이스 도달 횟수를 세려 했다.

```cpp
int fib(int n) {
    int count = 0;          // 지역 변수로 선언
    if(n == 1 || n == 2) {
        count++;            // 이 count는 현재 스택 프레임에만 존재
        return 1;
    }
    return fib(n-1) + fib(n-2);
}
```

재귀 호출마다 새 스택 프레임이 생기므로 `count`는 매번 0으로 초기화되어 누적이 불가능하다.

### 2. 핵심 아이디어

**코드1 (재귀 베이스 케이스 횟수):**

재귀 호출 전반에 걸쳐 값을 누적하려면 **전역 변수**를 사용해야 한다.

```cpp
int fib_count = 0;          // 전역 변수: 모든 재귀 호출이 공유

int fib(int n) {
    if(n == 1 || n == 2) {
        fib_count++;        // 베이스 케이스 도달마다 전역 변수 증가
        return 1;
    }
    return fib(n-1) + fib(n-2);
}
```

재귀 트리에서 베이스 케이스(`n=1` or `n=2`)는 모두 리프 노드이며, 각 리프는 `1`을 반환한다. `fib(n)`의 최종값이 이 `1`들의 합이므로, **베이스 케이스 호출 횟수 = fib(n)** 이 된다.

| n | fib(n) | 코드1 실행 횟수 |
|---|--------|------------|
| 1 | 1 | 1 |
| 2 | 1 | 1 |
| 3 | 2 | 2 |
| 4 | 3 | 3 |
| 5 | 5 | **5** |
| 30 | 832040 | **832040** |

**코드2 (DP 반복 횟수):**

`for i = 3 to n` 루프가 실행되는 횟수 = `n - 2` (단, n >= 3일 때).

| n | 코드2 실행 횟수 |
|---|------------|
| 5 | 5 - 2 = **3** |
| 30 | 30 - 2 = **28** |

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: 재귀 `O(2^n)`, DP `O(n)`
- 공간 복잡도: 재귀 `O(n)` (콜스택), DP `O(n)`

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int N;

int fib_count = 0;
int fib(int n) {
    if(n == 1 || n == 2) {
        fib_count++;
        return 1;
    }
    else return (fib(n-1) + fib(n-2));
}

vector<int> f;
int fibonacci(int n) {
    int count = 0;
    if (n == 1 || n == 2) return f[1];

    for (int i = 3; i < n+1; i++) {
        f[i] = f[i-1] + f[i-2];
        count++;
    }

    return count;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> N;
    f.assign(41, 0);
    f[1] = 1; f[2] = 1;
    fib(N);

    cout << fib_count << " " << fibonacci(N);

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|

|1차|미출력|재귀 내 지역 변수로 카운트 시도 → 스택 프레임마다 초기화되어 누적 불가|

### 상세 원인

재귀 함수에서 지역 변수는 호출마다 독립적인 스택 프레임에 존재한다.

```cpp
// 잘못된 코드: count가 각 호출마다 0으로 초기화됨
int fib(int n) {
    int count = 0;          // ← 매 호출마다 새로 생성
    if(n == 1 || n == 2) {
        count++;
        return 1;           // count는 이 스택 프레임과 함께 소멸
    }
    ...
}

// 올바른 코드: 전역 변수로 모든 호출이 같은 변수를 공유
int fib_count = 0;
int fib(int n) {
    if(n == 1 || n == 2) {
        fib_count++;        // ← 전역 변수이므로 누적됨
        return 1;
    }
    ...
}
```

### 개선점

- 재귀 호출 전반에 걸쳐 상태를 누적할 때는 지역 변수 대신 **전역 변수** 또는 **참조 전달(`int& count`)** 을 사용한다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 재귀 함수에서 카운트 누적하기

재귀 호출 간 값을 누적하려면 전역 변수나 참조 전달을 사용한다. 지역 변수는 각 스택 프레임에 귀속되어 호출 간 공유되지 않는다.

```cpp
// 방법1: 전역 변수
int count = 0;
void recur(int n) {
    if (base_case) { count++; return; }
    recur(n-1);
}

// 방법2: 참조 전달
void recur(int n, int& count) {
    if (base_case) { count++; return; }
    recur(n-1, count);
}
```

### 2. 재귀 피보나치 베이스 케이스 호출 횟수 = fib(n)

재귀 트리의 모든 리프는 베이스 케이스(`return 1`)이고, fib(n)은 이 1들의 합이므로 베이스 케이스 호출 횟수는 정확히 fib(n)과 같다. 재귀가 얼마나 비효율적인지 직관적으로 보여주는 패턴이다.
