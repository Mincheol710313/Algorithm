## 📝 문제 설명

```text
효주는 포도주 시식회에 갔다. 그 곳에 갔더니, 테이블 위에 다양한 포도주가 들어있는 포도주 잔이 일렬로 놓여 있었다. 효주는 포도주 시식을 하려고 하는데, 여기에는 다음과 같은 두 가지 규칙이 있다.

1. 포도주 잔을 선택하면 그 잔에 들어있는 포도주는 모두 마셔야 하고, 마신 후에는 원래 위치에 다시 놓아야 한다.
2. 연속으로 놓여 있는 3잔을 모두 마실 수는 없다.

효주는 될 수 있는 대로 많은 양의 포도주를 맛보기 위해서 어떤 포도주 잔을 선택해야 할지 고민하고 있다. 1부터 n까지의 번호가 붙어 있는 n개의 포도주 잔이 순서대로 테이블 위에 놓여 있고, 각 포도주 잔에 들어있는 포도주의 양이 주어졌을 때, 효주를 도와 가장 많은 양의 포도주를 마실 수 있도록 하는 프로그램을 작성하시오. 

예를 들어 6개의 포도주 잔이 있고, 각각의 잔에 순서대로 6, 10, 13, 9, 8, 1 만큼의 포도주가 들어 있을 때, 첫 번째, 두 번째, 네 번째, 다섯 번째 포도주 잔을 선택하면 총 포도주 양이 33으로 최대로 마실 수 있다.
```

### 입력
```text
첫째 줄에 포도주 잔의 개수 n이 주어진다. (1 ≤ n ≤ 10,000) 둘째 줄부터 n+1번째 줄까지 포도주 잔에 들어있는 포도주의 양이 순서대로 주어진다. 포도주의 양은 1,000 이하의 음이 아닌 정수이다.

6
6
10
13
9
8
1
```

### 출력
```text
첫째 줄에 최대로 마실 수 있는 포도주의 양을 출력한다.

33
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

`dp[i]`를 i번째 잔까지 고려했을 때 마실 수 있는 최대 포도주 양으로 정의했다.

처음에는 i번째 잔을 **반드시 마신다**는 전제 하에 케이스를 나눴다:

```
dp[0] = arr[0]
dp[1] = arr[1] + dp[0]
i >= 2: dp[i] = max(dp[i-2] + arr[i], arr[i-1] + arr[i])
```

### 2. 핵심 아이디어

`dp[i]`의 정의가 "i번째까지의 최대값"이므로, **i번째 잔을 건너뛰는 경우**도 반드시 포함해야 한다.

i번째 잔 기준으로 가능한 케이스는 3가지다:

| 케이스 | 설명 | 점화식 |
|---|---|---|
| i번 건너뜀 | i-1까지의 최적값 유지 | `dp[i-1]` |
| i-1번 건너뜀 | i번만 마심 | `dp[i-2] + arr[i]` |
| i-2번 건너뜀 | i-1, i번 연속 마심 | `dp[i-3] + arr[i-1] + arr[i]` |

또한 `dp[i-3]`을 사용하므로 Base Condition을 `dp[0]`, `dp[1]`, `dp[2]` 모두 직접 설정해야 한다.

**예시: `arr = [6, 10, 13, 9, 8, 1]`**

| i | arr[i] | dp[i-1] | dp[i-2]+arr[i] | dp[i-3]+arr[i-1]+arr[i] | dp[i] |
|---|--------|---------|----------------|--------------------------|-------|
| 0 | 6      | -       | -              | -                        | 6     |
| 1 | 10     | -       | -              | -                        | 16    |
| 2 | 13     | -       | -              | -                        | 23    |
| 3 | 9      | 23      | 16+9=25        | 6+13+9=**28**            | 28    |
| 4 | 8      | 28      | 23+8=31        | 16+9+8=**33**            | 33    |
| 5 | 1      | **33**  | 28+1=29        | 23+8+1=32                | 33    |

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(N)`

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;
    vector<int> grape(N, 0);
    for (int i = 0; i < N; i++) {
        cin >> grape[i];
    }

    vector<int> dp(N, 0);
    dp[0] = grape[0];
    dp[1] = grape[0] + grape[1];
    dp[2] = max(grape[0] + grape[1], max(grape[0] + grape[2], grape[1] + grape[2]));

    for (int i = 3; i < N; i++) {
        dp[i] = max(dp[i-1], max(dp[i-2] + grape[i], dp[i-3] + grape[i-1] + grape[i]));
    }

    cout << dp[N-1];

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차 (점화식 오류)|틀림|i번 건너뜀 케이스와 dp[i-3] 누락|

### 상세 원인

처음 작성한 점화식:
```cpp
// i번을 반드시 마신다고 가정하고, 두 케이스만 고려
dp[i] = max(dp[i-2] + arr[i], arr[i-1] + arr[i]);
```

올바른 점화식:
```cpp
// i번 건너뛰는 케이스 + dp[i-3] 반영
dp[i] = max(dp[i-1], max(dp[i-2] + grape[i], dp[i-3] + grape[i-1] + grape[i]));
```

두 가지 케이스가 빠졌다:
1. **`dp[i-1]`**: i번을 건너뛰는 경우 — dp의 정의상 반드시 포함해야 한다
2. **`dp[i-3] + arr[i-1] + arr[i]`**: `arr[i-1] + arr[i]`만 쓰면 i-3 이전의 최적값이 반영되지 않아 과소평가된다

### 개선점

- dp[i]를 "i번째까지의 최대값"으로 정의하면 **건너뛰는 케이스(`dp[i-1]`)를 항상 포함**해야 한다
- `dp[i-3]`처럼 3칸 이상 앞을 참조하면 Base Condition도 그만큼 더 설정해야 한다

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. dp 정의에서 "건너뜀" 케이스 포함

dp[i]를 "i번째까지 고려한 최대값"으로 정의하면, i번을 선택하지 않는 경우도 케이스에 포함해야 한다.

```cpp
dp[i] = max(dp[i-1], ...);  // i번 건너뜀 케이스 항상 포함
```

### 2. 점화식에서 참조 범위만큼 Base Condition 설정

점화식에서 `dp[i-k]`를 사용하면 `dp[0] ~ dp[k-1]`을 모두 직접 초기화해야 한다.

```cpp
// dp[i-3]을 쓰므로 0, 1, 2 모두 설정
dp[0] = grape[0];
dp[1] = grape[0] + grape[1];
dp[2] = max(grape[0]+grape[1], max(grape[0]+grape[2], grape[1]+grape[2]));
```
