## 📝 문제 설명

```text
수열 A가 주어졌을 때, 가장 긴 감소하는 부분 수열을 구하는 프로그램을 작성하시오.

예를 들어, 수열 A = {10, 30, 10, 20, 20, 10} 인 경우에 가장 긴 감소하는 부분 수열은 A = {10, 30, 10, 20, 20, 10}  이고, 길이는 3이다.
```

### 입력
```text
첫째 줄에 수열 A의 크기 N (1 ≤ N ≤ 1,000)이 주어진다.
둘째 줄에는 수열 A를 이루고 있는 Ai가 주어진다. (1 ≤ Ai ≤ 1,000)

6
10 30 10 20 20 10
```

### 출력
```text
첫째 줄에 수열 A의 가장 긴 감소하는 부분 수열의 길이를 출력한다.

3
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

LIS(가장 긴 증가하는 부분 수열)와 동일한 구조로, 비교 방향만 반대로 바꾸면 된다고 생각했다.

dp 정의를 처음에 "i번째**까지** 감소하는 부분 수열의 원소 개수"라고 했는데, 이렇게 정의하면 어떤 원소가 수열의 끝인지 특정할 수 없어서 점화식을 세우기 어렵다.

### 2. 핵심 아이디어

**DP 정의:** `dp[i]` = **i번째 원소로 끝나는** 가장 긴 감소하는 부분 수열의 길이

"i번째까지"가 아니라 "i번째로 끝나는"으로 정의해야 j → i 전이가 명확해진다.

**점화식:**
```
dp[i] = max(dp[j] + 1)  (0 ≤ j < i이고 arr[j] > arr[i]인 모든 j에 대해)
dp[i]의 초기값 = 1  (자기 자신 하나로 길이 1)
```

**예시: A = {10, 30, 10, 20, 20, 10}**

| i | arr[i] | 조건을 만족하는 j (arr[j] > arr[i]) | dp[i] |
|---|--------|--------------------------------------|-------|
| 0 | 10     | 없음                                  | 1     |
| 1 | 30     | 없음                                  | 1     |
| 2 | 10     | j=1 (30 > 10) → dp[1]+1=2            | 2     |
| 3 | 20     | j=1 (30 > 20) → dp[1]+1=2            | 2     |
| 4 | 20     | j=1 (30 > 20) → dp[1]+1=2            | 2     |
| 5 | 10     | j=1 (30>10), j=2 (10=10 불가), j=3 (20>10) → max(2,2)+1=3 | 3 |

→ dp 최댓값 = **3**

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N²)` — 이중 반복문 (i: 0~N, j: 0~i)
- 공간 복잡도: `O(N)` — dp 배열, arr 배열 각각 N

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;

    vector<int> arr(N, 0);
    for (int i = 0; i < N; i++) {
        cin >> arr[i];
    }

    vector<int> dp(N, 0);
    for (int i = 0; i < N; i++) {
        dp[i] = 1;
        for (int j = 0; j < i; j++) {
            if (arr[i] < arr[j]) dp[i] = max(dp[i], dp[j] + 1);
        }
    }

    int result = dp[0];
    for (int i = 1; i < N; i++) {
        result = max(result, dp[i]);
    }

    cout << result;
    return 0;
}
```

### 개선점

처음 DP를 정의할 때 **"i번째까지"와 "i번째로 끝나는"을 혼동**하지 않도록 주의해야 한다.

| 잘못된 정의 | 올바른 정의 |
|---|---|
| `dp[i]` = i번째**까지** 감소하는 부분 수열의 원소 개수 | `dp[i]` = i번째**로 끝나는** 가장 긴 감소하는 부분 수열의 길이 |

"까지"로 정의하면 끝 원소가 i인지 보장이 없어서 `dp[j]` → `dp[i]` 전이를 구성할 수 없다.  
"로 끝나는"으로 정의해야 j가 i의 바로 앞 원소임을 보장하고, 전이 조건(`arr[j] > arr[i]`)을 명확히 세울 수 있다.

-----

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. LDS는 LIS의 비교 방향만 뒤집은 것

LIS: `arr[j] < arr[i]`이면 `dp[i] = max(dp[i], dp[j] + 1)`  
LDS: `arr[j] > arr[i]`이면 `dp[i] = max(dp[i], dp[j] + 1)`

구조가 완전히 동일하고 비교 연산자만 반대이므로, LIS를 이해하면 LDS는 바로 적용할 수 있다.

### 2. 부분 수열 DP의 정의 원칙

부분 수열 길이를 구하는 DP는 항상 **"i번째로 끝나는"** 으로 정의한다.  
이렇게 해야 이전 상태 `dp[j]`에서 `dp[i]`로의 전이 조건을 명확히 세울 수 있다.

```cpp
dp[i] = 1;  // 자기 자신으로만 구성된 부분 수열
for (int j = 0; j < i; j++) {
    if (/* 전이 조건 */) dp[i] = max(dp[i], dp[j] + 1);
}
```

최종 답은 `dp` 배열 전체의 최댓값이다 (`*max_element(dp.begin(), dp.end())`).
