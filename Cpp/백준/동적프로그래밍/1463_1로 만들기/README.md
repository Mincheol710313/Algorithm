## 📝 문제 설명

```text
정수 X에 사용할 수 있는 연산은 다음과 같이 세 가지 이다.

1. X가 3으로 나누어 떨어지면, 3으로 나눈다.
2. X가 2로 나누어 떨어지면, 2로 나눈다.
3. 1을 뺀다.

정수 N이 주어졌을 때, 위와 같은 연산 세 개를 적절히 사용해서 1을 만들려고 한다.
연산을 사용하는 횟수의 최솟값을 출력하시오.
```

### 입력
```text
10
```

### 출력
```text
3
```

-----

## 🧠 풀이 과정

### 1. 처음 접근 방법

Bottom-Up 방식으로 접근했다. `dp[i]`를 "i를 1로 만드는 최소 연산 횟수"로 정의하고,  
작은 수부터 차례대로 채워나갔다.

처음에는 세 연산을 `else if`로 연결해 하나만 선택했는데, 이게 문제였다.

### 2. 핵심 아이디어

**세 가지 경우를 모두 따져 최솟값을 골라야 한다.**

`i = 6`을 예로 들면:
- 1을 빼는 경우: `dp[5] + 1 = 4`
- 2로 나누는 경우: `dp[3] + 1 = 2`
- 3으로 나누는 경우: `dp[2] + 1 = 2`

→ 최솟값 `2` 선택

**점화식 (Bottom-Up / Top-Down 공통):**
```
dp[n] = dp[n-1] + 1                          // 항상 가능
dp[n] = min(dp[n], dp[n/2] + 1)  if n % 2 == 0
dp[n] = min(dp[n], dp[n/3] + 1)  if n % 3 == 0
```

**dp 테이블 (일부):**

| i | dp[i] | 선택한 연산 |
|---|-------|------------|
| 1 | 0     | (시작)      |
| 2 | 1     | ÷2          |
| 3 | 1     | ÷3          |
| 4 | 2     | ÷2 → ÷2     |
| 5 | 3     | -1 → ÷2 → ÷2 |
| 6 | 2     | ÷3 또는 ÷2  |
| 10| 3     | ÷2 → -1 → ÷3 |

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)` — 각 상태는 최대 한 번만 계산
- 공간 복잡도: `O(N)` — dp 배열 크기

## 💻 최종 코드

### Bottom-Up (Tabulation)

```cpp
vector<int> dp(1000001, -1);

void bottom_up_sol() {
    dp[1] = 0;

    for (int i = 2; i <= 1000000; i++) {
        dp[i] = dp[i-1] + 1;
        if (i % 2 == 0) dp[i] = min(dp[i], dp[i/2] + 1);
        if (i % 3 == 0) dp[i] = min(dp[i], dp[i/3] + 1);
    }
}
```

### Top-Down (Memoization)

```cpp
vector<int> dp(1000001, -1);

int top_down_sol(int n) {
    if (n == 1) return 0;
    if (dp[n] != -1) return dp[n];

    dp[n] = top_down_sol(n-1) + 1;
    if (n % 2 == 0) dp[n] = min(dp[n], top_down_sol(n/2) + 1);
    if (n % 3 == 0) dp[n] = min(dp[n], top_down_sol(n/3) + 1);

    return dp[n];
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|틀림|`else if`로 연산 하나만 선택|
|2차|틀림|벡터 크기 부족 (N=1,000,000일 때 Out of Bounds)|
|3차|틀림|Top-Down에서 `dp[n-1]` 직접 참조 (미계산 상태)|

### 상세 원인

**1차: else if 문제**

```cpp
// 잘못됨 — 세 조건 중 하나만 평가
if (i % 3 == 0) dp[i] = dp[i/3] + 1;
else if (i % 2 == 0) dp[i] = dp[i/2] + 1;
else dp[i] = dp[i-1] + 1;

// 올바름 — 모든 경우 따진 후 최솟값 선택
dp[i] = dp[i-1] + 1;
if (i % 2 == 0) dp[i] = min(dp[i], dp[i/2] + 1);
if (i % 3 == 0) dp[i] = min(dp[i], dp[i/3] + 1);
```

**2차: 배열 크기 Off-by-one**

```cpp
// 잘못됨 — dp[1000000] 접근 시 Out of Bounds
vector<int> dp(1000000, -1);

// 올바름
vector<int> dp(1000001, -1);
```

**3차: Top-Down에서 dp 배열 직접 참조**

Bottom-Up은 순서대로 채우므로 `dp[i-1]`이 항상 계산된 상태지만,  
Top-Down은 호출된 것만 계산하므로 `dp[n-1]`이 아직 `-1`일 수 있다.

```cpp
// 잘못됨 — dp[n-1]이 -1인 채로 참조될 수 있음
dp[n] = dp[n-1] + 1;

// 올바름 — 재귀 호출로 계산을 보장
dp[n] = top_down_sol(n-1) + 1;
```

### 개선점

- 여러 경우 중 최솟값을 구할 땐 `else if`가 아니라 독립적인 `if`로 모두 검토한다.
- 배열 크기는 **최댓값 + 1**로 잡는다.
- Top-Down에서는 dp 배열을 직접 참조하지 말고, 항상 **재귀 함수를 통해** 값을 가져온다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. Bottom-Up vs Top-Down 구현 차이

같은 점화식이라도 구현 방식이 다르다.

| | Bottom-Up | Top-Down |
|--|--|--|
| **순서** | 작은 것부터 순서대로 | 필요한 것만 재귀 호출 |
| **dp 참조** | `dp[i-1]` 직접 참조 가능 (이미 계산됨) | 반드시 `func(n-1)` 재귀 호출로 참조 |
| **base case** | 루프 전에 `dp[1] = 0` | 함수 시작에 `if (n == 1) return 0` |

```cpp
// Bottom-Up: dp 직접 참조 OK
dp[i] = dp[i-1] + 1;

// Top-Down: 재귀 호출로 참조해야 함
dp[n] = top_down_sol(n-1) + 1;
```

### 2. 여러 전환이 있을 때 else if 금지

최솟값을 구하는 DP에서는 가능한 모든 전환을 독립적인 `if`로 검토한다.

### 3. 배열 크기는 최댓값 + 1

```cpp
vector<int> dp(N + 1, -1); // dp[0] ~ dp[N] 모두 유효
```
