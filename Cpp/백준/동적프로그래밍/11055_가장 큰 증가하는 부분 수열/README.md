## 📝 문제 설명

```text
수열 A가 주어졌을 때, 그 수열의 증가하는 부분 수열 중에서 합이 가장 큰 것을 구하는 프로그램을 작성하시오.

예를 들어, 수열 A = {1, 100, 2, 50, 60, 3, 5, 6, 7, 8} 인 경우에 합이 가장 큰 증가하는 부분 수열은 A = {1, 100, 2, 50, 60, 3, 5, 6, 7, 8} 이고, 합은 113이다.
```

### 입력
```text
첫째 줄에 수열 A의 크기 N (1 ≤ N ≤ 1,000)이 주어진다.
둘째 줄에는 수열 A를 이루고 있는 Ai가 주어진다. (1 ≤ Ai ≤ 1,000)

10
1 100 2 50 60 3 5 6 7 8
```

### 출력
```text
첫째 줄에 수열 A의 합이 가장 큰 증가하는 부분 수열의 합을 출력한다.

113
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

11053 LIS(가장 긴 증가하는 부분 수열) 문제와 구조가 동일하다는 것을 바로 파악했다.
DP 배열을 어떻게 정의하느냐가 핵심이라는 점을 인식하고, **dp[i] = arr[i]로 끝나는 증가 부분 수열의 합의 최대값**으로 정의했다.

### 2. 핵심 아이디어

LIS 길이 문제의 점화식 `dp[i] = max(dp[i], dp[j] + 1)` 에서 `+1` 을 `+arr[i]` 로 바꾸면 합 버전이 된다.

- 초기값: `dp[i] = arr[i]` (자기 자신 하나만으로 이루어진 부분 수열)
- 점화식: `arr[i] > arr[j]` (j < i) 인 경우 → `dp[i] = max(dp[i], dp[j] + arr[i])`
- 최종 답: `max(dp[0], ..., dp[N-1])`

**`arr = {1, 100, 2, 50, 60}` 단계별 예시**

| i | arr[i] | 갱신 과정 | dp[i] |
|---|--------|----------|-------|
| 0 | 1      | 초기값    | 1     |
| 1 | 100    | dp[0]+100 = **101** | 101 |
| 2 | 2      | dp[0]+2 = **3** | 3 |
| 3 | 50     | dp[0]+50=51, dp[2]+50=**53** | 53 |
| 4 | 60     | dp[2]+60=63, dp[3]+60=**113** | 113 |

→ `max(dp) = 113` (수열: [1, 2, 50, 60])

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N²)` — 이중 반복문
- 공간 복잡도: `O(N)` — dp 배열

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;

    vector<int> arr(N, 0);
    for (int i = 0; i < N; i++) cin >> arr[i];

    vector<int> dp(N, 0);
    for (int i = 0; i < N; i++) {
        dp[i] = arr[i];
        for (int j = 0; j < i; j++) {
            if (arr[i] > arr[j]) dp[i] = max(dp[i], dp[j] + arr[i]);
        }
    }

    int result = dp[0];
    for (int i = 1; i < N; i++) result = max(result, dp[i]);

    cout << result;

    return 0;
}
```

### 개선점

- `dp[i]` 초기값을 `arr[i]` 로 설정하는 것이 핵심이다. `0`으로 초기화하면 앞에 연결할 원소가 없는 경우 arr[i] 자체가 누락된다.
- 최종 답은 `dp[N-1]`이 아니라 `max(dp)` 임에 주의한다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. DP 배열 정의가 곧 점화식이다

DP 문제에서 `dp[i]`를 무엇으로 정의하느냐에 따라 점화식이 자연스럽게 결정된다. 먼저 "dp[i]가 무엇을 의미하는지"를 명확히 정하고 점화식을 유도하는 순서로 접근한다.

### 2. LIS 변형 패턴

LIS 계열 문제는 dp 정의와 점화식의 `+1` 부분만 바꾸면 다양한 변형을 처리할 수 있다.

| 문제 | dp[i] 정의 | 점화식 |
|------|-----------|--------|
| 11053 가장 긴 증가 부분 수열 (길이) | arr[i]로 끝나는 LIS 길이 | `dp[j] + 1` |
| 11055 가장 큰 증가 부분 수열 (합)  | arr[i]로 끝나는 부분 수열의 합 | `dp[j] + arr[i]` |
