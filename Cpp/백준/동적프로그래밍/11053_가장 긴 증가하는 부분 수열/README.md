## 📝 문제 설명

```text
수열 A가 주어졌을 때, 가장 긴 증가하는 부분 수열을 구하는 프로그램을 작성하시오.

예를 들어, 수열 A = {10, 20, 10, 30, 20, 50} 인 경우에 가장 긴 증가하는 부분 수열은 A = {10, 20, 10, 30, 20, 50} 이고, 길이는 4이다.
```

### 입력
```text
첫째 줄에 수열 A의 크기 N (1 ≤ N ≤ 1,000)이 주어진다.
둘째 줄에는 수열 A를 이루고 있는 Ai가 주어진다. (1 ≤ Ai ≤ 1,000)

6
10 20 10 30 20 50
```

### 출력
```text
첫째 줄에 수열 A의 가장 긴 증가하는 부분 수열의 길이를 출력한다.

4
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

`dp[i] = max(dp[i-1], arr[i])` 로 누적 최대값 배열을 만든 뒤, dp 배열 내 **서로 다른 숫자의 개수**를 세면 LIS 길이가 나올 것이라고 생각했다.

`arr = [10, 20, 10, 30, 20, 50]` 에서는 우연히 맞는 것처럼 보인다.

```
dp = [10, 20, 20, 30, 30, 50]
서로 다른 값: {10, 20, 30, 50} → 4  (정답과 일치)
```

하지만 이 접근은 틀렸다.

### 2. 핵심 아이디어

`dp[i] = arr[i]로 끝나는 LIS의 길이` 로 정의한다.

- 초기값: `dp[i] = 1` (자기 자신만으로도 길이 1짜리 수열)
- 점화식: `arr[j] < arr[i]` 인 모든 `j < i` 에 대해 `dp[i] = max(dp[i], dp[j] + 1)`
- 최종 답: `max(dp[0], dp[1], ..., dp[N-1])`

**`arr = [10, 20, 10, 30, 20, 50]` 단계별 예시**

| i | arr[i] | 비교 대상 (j < i, arr[j] < arr[i]) | dp[i] |
|---|--------|----------------------------------|-------|
| 0 | 10     | 없음                               | 1     |
| 1 | 20     | j=0 (10<20): dp[0]+1=2           | 2     |
| 2 | 10     | 없음 (앞의 값이 모두 ≥ 10)            | 1     |
| 3 | 30     | j=0,1,2 → max(2,3,2)=3          | 3     |
| 4 | 20     | j=0,2: max(2,2)=2               | 2     |
| 5 | 50     | j=0~4 → max(2,3,2,4,3)=4       | 4     |

→ `max(dp) = 4`

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N²)` — 이중 반복문
- 공간 복잡도: `O(N)` — dp 배열

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;

    vector<int> arr(N);
    for (int i = 0; i < N; i++) cin >> arr[i];

    vector<int> dp(N, 1);

    for (int i = 1; i < N; i++) {
        for (int j = 0; j < i; j++) {
            if (arr[j] < arr[i]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
    }

    cout << *max_element(dp.begin(), dp.end()) << "\n";

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|오답|`dp[i] = max(dp[i-1], arr[i])` 후 distinct 카운트|

### 상세 원인

`arr = [3, 1, 2]` 반례:

```
// 잘못된 접근
dp[0] = 3
dp[1] = max(3, 1) = 3
dp[2] = max(3, 2) = 3
dp = [3, 3, 3] → distinct 개수 = 1  (오답)

// 올바른 접근
dp[0] = 1  (수열: [3])
dp[1] = 1  (수열: [1], 3 >= 1이라 갱신 없음)
dp[2] = 2  (1 < 2이므로 dp[1]+1=2)
max(dp) = 2  (정답: [1, 2])
```

누적 최대값 방식은 **앞에 큰 수가 나온 이후 작은 수부터 시작하는 증가 수열**을 전혀 감지하지 못한다.

### 개선점

- dp[i]는 "i번째 원소로 끝나는 LIS 길이"로 정의해야 한다.
- 최종 답은 `dp[N-1]`이 아니라 `max(dp)` 임에 주의한다.

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. LIS 기본 점화식

`dp[i]` = arr[i]로 끝나는 LIS 길이, 최종 답은 dp 전체의 최대값

```cpp
vector<int> dp(N, 1);
for (int i = 1; i < N; i++)
    for (int j = 0; j < i; j++)
        if (arr[j] < arr[i])
            dp[i] = max(dp[i], dp[j] + 1);
int ans = *max_element(dp.begin(), dp.end());
```

### 2. "뒤에서 시작하는 수열"을 놓치지 않으려면

누적 최대·최소 등 단방향 DP는 앞 원소에 의존하므로, **이전의 모든 원소를 조건부로 탐색**하는 이중 반복문이 필요하다.
