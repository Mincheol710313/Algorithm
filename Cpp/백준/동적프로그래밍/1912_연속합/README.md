## 📝 문제 설명

```text
n개의 정수로 이루어진 임의의 수열이 주어진다. 우리는 이 중 연속된 몇 개의 수를 선택해서 구할 수 있는 합 중 가장 큰 합을 구하려고 한다. 단, 수는 한 개 이상 선택해야 한다.

예를 들어서 10, -4, 3, 1, 5, 6, -35, 12, 21, -1 이라는 수열이 주어졌다고 하자. 여기서 정답은 12+21인 33이 정답이 된다.
```

### 입력
```text
첫째 줄에 정수 n(1 ≤ n ≤ 100,000)이 주어지고 둘째 줄에는 n개의 정수로 이루어진 수열이 주어진다. 수는 -1,000보다 크거나 같고, 1,000보다 작거나 같은 정수이다.

10
10 -4 3 1 5 6 -35 12 21 -1
```

### 출력
```text
첫째 줄에 답을 출력한다.

33
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

`dp[i]`를 "i번째까지 고려한 최대 연속합"으로 정의하려 했다.

그런데 이 정의로는 점화식을 세울 수 없다. `dp[i-1]`이 arr[i-1]에서 끝났는지, 더 앞에서 끝났는지 알 수 없어서 arr[i]를 이어붙일 수 있는지 판단이 불가능하기 때문이다.

### 2. 핵심 아이디어

`dp[i]`를 **"i번째 원소에서 끝나는 연속합의 최대값"** 으로 정의한다.

이 정의 덕분에:
- `dp[i-1]`은 반드시 arr[i-1]을 포함하고 arr[i-1]에서 끝나는 수열 → `dp[i-1] + arr[i]`는 자동으로 연속이 보장
- arr[i]를 반드시 포함하므로 선택지는 딱 2가지

```
dp[i] = max(dp[i-1] + arr[i], arr[i])
         ↑ 이전 연속합에 이어붙임   ↑ arr[i] 혼자 새로 시작
```

dp[i]가 항상 arr[i]를 포함하기 때문에, 최종 답은 `dp[N-1]`이 아니라 **`dp[0] ~ dp[N-1]` 중 최대값**이다.

**예시: `arr = [10, -4, 3, 1, 5, 6, -35, 12, 21, -1]`**

| i | arr[i] | dp[i-1]+arr[i] | arr[i] 단독 | dp[i] |
|---|--------|----------------|-------------|-------|
| 0 | 10     | -              | 10          | **10** |
| 1 | -4     | 10+(-4)=6      | -4          | **6** |
| 2 | 3      | 6+3=9          | 3           | **9** |
| 3 | 1      | 9+1=10         | 1           | **10** |
| 4 | 5      | 10+5=15        | 5           | **15** |
| 5 | 6      | 15+6=21        | 6           | **21** |
| 6 | -35    | 21+(-35)=-14   | -35         | **-14** |
| 7 | 12     | -14+12=-2      | 12          | **12** |
| 8 | 21     | 12+21=33       | 21          | **33** |
| 9 | -1     | 33+(-1)=32     | -1          | **32** |

`max(dp) = 33` → 정답

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(N)`

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;
    vector<int> arr(N);
    for (int i = 0; i < N; i++) cin >> arr[i];

    vector<int> dp(N);
    dp[0] = arr[0];

    for (int i = 1; i < N; i++) {
        dp[i] = max(dp[i-1] + arr[i], arr[i]);
    }

    cout << *max_element(dp.begin(), dp.end());

    return 0;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차 (설계 오류)|틀림|dp 정의를 "i번째까지의 최대값"으로 잡아 점화식 성립 불가|

### 상세 원인

잘못된 정의:
```
dp[i] = "i번째까지 고려한 연속합 최대값"
```

이 정의로는 `dp[i-1]`이 어디서 끝났는지 알 수 없어서 arr[i]를 이어붙일 수 있는지 판단 불가능하다.

올바른 정의:
```
dp[i] = "i번째 원소에서 끝나는 연속합 최대값"  → arr[i]를 반드시 포함
```

이렇게 하면 `dp[i-1]`은 항상 arr[i-1]에서 끝난다는 것이 보장되어 연속성이 유지된다.

### 개선점

- "연속"이라는 제약이 있는 문제는 dp를 **"특정 위치에서 끝나는"** 으로 정의하면 점화식이 자연스럽게 성립된다
- 이 경우 최종 답은 `dp[N-1]`이 아니라 `max(dp[0]...dp[N-1])`임을 주의

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. "i에서 끝나는" dp 정의 패턴

연속성 제약이 있는 문제에서는 dp[i]를 **"i번째에서 끝나는 최적값"** 으로 정의하면 이전 dp 값과의 연결이 자연스럽게 보장된다.

```cpp
dp[i] = max(dp[i-1] + arr[i], arr[i]);  // 이어붙이기 vs 새로 시작
```

### 2. "i까지 고려" vs "i에서 끝나는" 정의 선택 기준

| 정의 | 언제 사용 | 최종 답 |
|------|-----------|---------|
| `dp[i]` = i번째까지 고려한 최적값 | 원소를 건너뛸 수 있는 경우 (포도주 시식 등) | `dp[N-1]` |
| `dp[i]` = i번째에서 끝나는 최적값 | 연속성이 필요한 경우 (연속합 등) | `max(dp[0]...dp[N-1])` |
