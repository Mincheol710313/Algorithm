## 📝 문제 설명

```text
N개의 정수로 이루어진 수열 A[1], A[2], …, A[N]이 있다. 이 수열에서 두 수를 골랐을 때(같은 수일 수도 있다),
그 차이가 M 이상이면서 제일 작은 경우를 구하는 프로그램을 작성하시오.
```

### 입력
```text
3 3
1
5
3
```

### 출력
```text
4
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

두 수의 **차이**를 구하는 문제라 정렬 후 투포인터를 떠올렸다. 처음에는 "두 수의 합"처럼 양쪽 끝에서 좁혀오는 방식을 생각했다.

- diff >= M이면 → left++, right-- 중 하나를 이동
- diff < M이면 → ?

하지만 diff < M일 때 차이를 **키울 방법이 없다**는 문제가 생겼다. 양쪽 끝에서 시작하면 이미 left는 최솟값, right는 최댓값 위치라서 창을 확장할 수 없기 때문이다.

**반례: `[1, 3, 5, 8, 10]`, M=5**

| 방식 | 경로 | 결과 |
|------|------|------|
| 양쪽에서 좁히기 (right--) | (1,10)=9 → (1,8)=7 → (1,5)=4 < 5 → 종료 | **7** (틀림) |
| 같은 방향 | 정상 탐색 후 (3,8)=5 발견 | **5** (정답) |

→ 양쪽에서 좁히기로는 `(3,8)` 쌍을 영원히 확인하지 못한다.

### 2. 핵심 아이디어

**정렬 후 두 포인터를 모두 왼쪽(0)에서 시작해 같은 방향으로 이동**한다.

| 상황 | 이유 | 동작 |
|------|------|------|
| diff < M | 차이가 너무 작다 → 창을 넓혀야 함 | right++ |
| diff > M | 조건 만족, 더 작은 값을 찾아야 함 | result 갱신, left++ |
| diff == M | 가능한 최솟값 → 즉시 종료 | result = M, break |

**단계별 포인터 변화 (예시: `[1, 3, 5]`, M=3)**

| 단계 | left | right | diff | 동작 |
|------|------|-------|------|------|
| 1 | 0 (1) | 0 (1) | 0 | diff<3 → right++ |
| 2 | 0 (1) | 1 (3) | 2 | diff<3 → right++ |
| 3 | 0 (1) | 2 (5) | 4 | diff>3 → result=4, left++ |
| 4 | 1 (3) | 2 (5) | 2 | diff<3 → right++ |
| 5 | - | right=3 ≥ N | - | 종료 |
| 최종 | | | | **result = 4** |

**핵심 관계:**
```
두 수의 합:  left → ← right   (양쪽에서 좁히기)
두 수의 차:  left →  right →  (같은 방향)
```

두 문제가 달라 보여도 같은 투포인터인데, diff를 **키우는 방향이 right++뿐**이라는 점이 방향을 결정한다.

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(N log N)` — 정렬 O(N log N) + 투포인터 O(N)
- 공간 복잡도: `O(N)` — 수열 저장 배열

## 💻 최종 코드

```cpp
int main() {
    int N, M;
    vector<int> arr;
    cin >> N >> M;
    arr.assign(N, 0);
    for (int i = 0; i < N; i++) cin >> arr[i];

    sort(arr.begin(), arr.end());

    int left = 0, right = 0;
    int diff = 0;
    int result = -1;

    while (right < N) {
        if (diff == M) {
            result = M;
            break;                              // M 이상 중 최솟값이므로 즉시 종료
        } else if (diff > M) {
            if (result == -1) result = diff;
            else result = min(diff, result);    // 최솟값 갱신
            left++;
            diff = arr[right] - arr[left];      // left 이동 후 diff 재계산
        } else {
            right++;
            if (right < N) diff = arr[right] - arr[left];
        }
    }

    cout << result;
}
```

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|

### 고민했던 부분

**1. 양쪽에서 좁히는 방식이 틀린 이유**

diff < M일 때 차이를 키워야 하는데, 양쪽 끝에서 시작하면 창 확장이 불가능하다.
left-- 또는 right++로 가야 하는데 이미 경계라서 막혀있다.

**2. diff 변수 하나만으로는 부족한 이유**

`diff`는 현재 창의 차이값이고, 루프를 돌면서 계속 바뀐다.
지금까지 찾은 **최솟값**을 기억하려면 `result`가 별도로 필요하다.

```cpp
// diff만 있으면 이전에 찾은 최솟값을 잃어버림
diff=7 → left++ → diff=5 → left++ → diff=3(< M) → right++ → diff=6
// 5가 최솟값이었는데 기억하지 못함

// result가 있어야 함
result = min(result, diff)  // 5를 기억
```

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 두 수의 합 vs 두 수의 차 — 포인터 방향이 다르다

```
두 수의 합 (목표 T):
  sum > T → right--   (합을 줄임)
  sum < T → left++    (합을 키움)
  → 양쪽 끝에서 서로 좁혀오는 방향

두 수의 차 (최소 M 이상):
  diff < M → right++  (차이를 키움, right만 확장 가능)
  diff >= M → left++  (차이를 줄이며 더 나은 답 탐색)
  → 두 포인터 모두 같은 방향(왼→오)으로 이동
```

### 2. diff == M이면 즉시 break

M 이상 중 최솟값을 구하는 문제이므로, 정확히 M이 나오는 순간 더 볼 필요가 없다.

```cpp
if (diff == M) { result = M; break; }
```

### 3. 현재값(diff)과 최솟값(result)은 항상 분리

투포인터에서 최솟값/최댓값을 구할 때, 루프 내 계산값과 정답 추적 변수는 반드시 분리한다.

```cpp
int diff = 0;     // 현재 창의 차이
int result = -1;  // 지금까지 찾은 최솟값
```
