## 📝 문제 설명

```text
갑옷은 두 개의 재료로 만드는데, 두 재료의 고유 번호의 합이 M이 되면 갑옷이 만들어진다.
N개의 재료와 M이 주어졌을 때, 만들 수 있는 갑옷의 수를 구하시오.
```

### 입력
```text
6
9
2 7 4 1 5 3
```

### 출력
```text
2
```

## 🧠 풀이 과정

### 0. 학습 방식

> ⚠️ 투 포인터를 처음 학습하는 날 개념을 참고하며 작성한 코드입니다.
> 블라인드 코딩이 아닌 학습 목적의 구현으로, 패턴 이해에 초점을 맞춰 정리합니다.
> 이전 문제(2003. 수들의 합 2)와 다른 **양방향 투포인터** 패턴입니다.

### 1. 처음 접근 방법

두 재료의 합이 M인 쌍을 찾는 문제로, 완전탐색 O(N²)을 먼저 떠올렸다.
하지만 투포인터 학습 중이었기 때문에, **정렬 후 양 끝에서 포인터를 좁혀가는** 방식을 적용했다.

### 2. 핵심 아이디어

배열을 **정렬**한 뒤, 양 끝에 포인터를 두고 합을 보며 이동한다.

| 조건 | 행동 |
|------|------|
| `sum == M` | count++, left++, right-- (둘 다 이동) |
| `sum < M`  | left++ (합을 키움) |
| `sum > M`  | right-- (합을 줄임) |

**예시: N=6, M=9, 배열=[2,7,4,1,5,3]**

```
정렬 후: [1, 2, 3, 4, 5, 7]
          ↑                ↑
        left=0          right=5

step1: 1+7=8  < 9  → left=1
step2: 2+7=9 == 9  → count=1, left=2, right=4
step3: 3+5=8  < 9  → left=3
step4: 4+5=9 == 9  → count=2, left=4, right=3
       left >= right → 종료

결과: 2
```

정렬 덕분에 합이 크면 오른쪽을 줄이고, 작으면 왼쪽을 키우는 판단이 항상 유효하다.

### 3. 시간/공간 복잡도 분析

- 시간 복잡도: `O(N log N)` — 정렬이 병목, 투포인터 탐색은 O(N)
- 공간 복잡도: `O(N)` — 재료 배열

## 💻 최종 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N, M;
    cin >> N >> M;

    vector<int> arr(N);
    for (int i = 0; i < N; i++) cin >> arr[i];

    sort(arr.begin(), arr.end());

    int left = 0, right = N - 1;
    int count = 0;

    while (left < right) {
        int sum = arr[left] + arr[right];
        if (sum == M) {
            count++;
            left++;
            right--;
        } else if (sum < M) {
            left++;
        } else {
            right--;
        }
    }

    cout << count;
    return 0;
}
```

## ❌ 틀린 이유 & 개선점

> 개념 학습 중 참고하며 작성해 제출 오류는 없었습니다.
> 처음 배울 때 헷갈렸던 포인트를 정리합니다.

| 헷갈린 부분 | 내용 |
|------------|------|
| 정렬 필수 여부 | 정렬 없이 양방향 포인터를 쓰면 합의 대소 판단이 무의미해짐 |
| sum == M일 때 한 쪽만 이동 | 두 포인터를 **모두** 이동해야 중복 쌍을 피할 수 있음 |
| `left < right` 조건 | `left == right`는 같은 재료를 두 번 쓰는 것이므로 제외 |

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 양방향 투포인터 (Two Sum 패턴)

정렬된 배열에서 두 원소의 합이 목표값인 쌍을 찾을 때 사용.

```cpp
sort(arr.begin(), arr.end());

int left = 0, right = N - 1;
while (left < right) {
    int sum = arr[left] + arr[right];
    if (sum == target)      { /* 정답 처리 */ left++; right--; }
    else if (sum < target)  { left++; }
    else                    { right--; }
}
```

**언제 쓰나:** 정렬된(or 정렬 가능한) 배열에서 **두 원소의 합**이 특정 값인 쌍 탐색.

### 2. 단방향 vs 양방향 투포인터 비교

| | 단방향 (같은 방향) | 양방향 (반대 방향) |
|--|--|--|
| 대표 문제 | 수들의 합 2 (2003) | 주몽 (1940) |
| 포인터 위치 | left=0, right=0 | left=0, right=N-1 |
| 목적 | 연속 구간의 합 | 두 원소의 합 |
| 정렬 필요 | 불필요 (양수 배열이면 됨) | 필수 |
| 종료 조건 | right >= N | left >= right |
