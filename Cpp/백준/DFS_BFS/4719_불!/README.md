## 📝 문제 설명

```text
지훈이는 미로에서 일을 한다. 지훈이를 미로에서 탈출하도록 도와주자!
미로에서의 지훈이의 위치와 불이 붙은 위치를 감안해서 지훈이가 불에 타기전에 탈출할 수 있는지의 여부,
그리고 얼마나 빨리 탈출할 수 있는지를 결정해야한다.
지훈이와 불은 매 분마다 한칸씩 수평 또는 수직으로 이동한다.
불은 각 지점에서 네 방향으로 확산된다.
지훈이는 미로의 가장자리에 접한 공간에서 탈출할 수 있다.
지훈이와 불은 벽이 있는 공간은 통과하지 못한다.
```

### 입력
```text
4 4
####
#JF#
#..#
#..#
```

### 출력
```text
3
```

## 🧠 풀이 과정

### 1. 처음 접근 방법

불과 지훈을 하나의 BFS 큐에 함께 넣고 `who` 플래그(0=지훈, 1=불)로 구분해서 동시에 처리하려 했다.
`grid`를 직접 `'J'`, `'F'`로 수정하며 이동을 표현했다.

이 접근의 문제점:
- 방문 체크가 없어서 지훈이 이전 위치가 `'.'`로 복원되면 다시 방문 가능 → BFS 큐가 무한히 커져 TLE
- 지훈과 불이 동시에 도달하는 셀에서 큐 삽입 순서에 따라 결과가 달라짐 → 정확성 보장 불가

### 2. 핵심 아이디어

**BFS를 두 단계로 분리한다.**

**1단계: 불 BFS (멀티 소스)**

모든 `'F'` 위치를 동시에 큐에 넣고 BFS를 돌려 `fire_dist[i][j]` (불이 `(i,j)`에 도달하는 시간)를 계산한다.
불이 닿지 않는 셀은 `-1` (INF)로 유지된다.

```
초기 상태 (fire_dist, -1은 미도달):
####        ####
#J0#   →    #J0#
#..#        #-1-#  ← 불이 벽에 막혀 못 옴
#..#        #-1-#
```

**2단계: 지훈 BFS**

`(next_row, next_col)`로 이동 가능 조건:

```
grid[nr][nc] != '#'              // 벽이 아님
&& jihun_dist[nr][nc] == -1      // 미방문
&& (fire_dist[nr][nc] == -1      // 불이 절대 안 오거나
    || jihun_dist[cur] + 1 < fire_dist[nr][nc])  // 지훈이 불보다 먼저 도달
```

동시 도달(`==`)은 이동 불가 — 지훈이 타는 경우이므로 **엄격한 부등호**(`<`)를 사용해야 한다.

**탈출 감지**: `inRange` 체크 전에 먼저 확인한다.

```cpp
if (!inRange(next_row, next_col)) {   // 격자 밖으로 나가면 탈출
    return jihun_dist[cur_row][cur_col] + 1;
}
```

경계 셀에 있다고 해서 탈출이 아니라, **격자 밖으로 실제로 이동**할 때 탈출이다.

### 3. 시간/공간 복잡도 분석

- 시간 복잡도: `O(R * C)` — 불 BFS O(RC) + 지훈 BFS O(RC)
- 공간 복잡도: `O(R * C)` — grid, fire_dist, jihun_dist 배열

## ❌ 틀린 이유 & 개선점

|시도|결과|원인|
|--|--|--|
|1차|TLE|불+지훈 혼합 BFS에서 방문 체크 없음 → 무한 루프|
|2차|오답|`dist == 0` visited 체크 추가로 TLE 해결. 그러나 동시 도달 처리 오류 + isExit 경계 누락|
|3차|모든 출력이 1|BFS 분리 후 탈출 조건에서 `!inRange` 대신 `inRange` 오타|

### 상세 원인

**1차: isExit 함수 버그**

```cpp
// 처음
bool isExit(int cr, int cc) { return cr == r || cc == c; }  // 절대 true 안 됨

// 수정 1 (오른쪽/아래만 체크)
bool isExit(int cr, int cc) { return cr == r-1 || cc == c-1; }  // top, left 누락

// 올바른 형태 (단, 이 함수 자체가 불필요했음)
bool isExit(int cr, int cc) { return cr == 0 || cr == r-1 || cc == 0 || cc == c-1; }
```

BFS는 `inRange`를 통과한 좌표만 큐에 넣으므로 `cr == r`은 절대 도달 불가.
탈출 감지는 `!inRange(next_row, next_col)` 방식이 정확하다.

**2차: 혼합 BFS의 동시 도달 문제**

불과 지훈을 같은 큐에서 처리하면, 동일 거리에서 큐 삽입 순서에 따라 지훈이 먼저 처리될 수 있다.
이 경우 불이 도달해야 할 셀을 지훈이 선점하여 불이 차단되는 잘못된 결과가 나온다.

**3차: !inRange 오타**

```cpp
// 버그: inRange이면 반환 → 지훈 시작 위치의 첫 이웃에서 즉시 0+1=1 반환
if (inRange(next_row, next_col)) return jihun_dist[cur_row][cur_col] + 1;

// 올바른 코드
if (!inRange(next_row, next_col)) return jihun_dist[cur_row][cur_col] + 1;
```

### 개선점

- 탈출 조건을 `isExit(border cell)` 방식이 아닌 `!inRange(next)` 방식으로 작성하면 실수를 줄일 수 있다
- 여러 주체(불, 지훈)가 동시에 움직이는 문제는 반드시 BFS를 분리해서 시간 비교로 처리한다

## 💡 배운 점 & 다음에 써먹을 패턴

### 1. 멀티 소스 BFS

시작점이 여러 개일 때 모두 큐에 넣고 한 번만 BFS를 돌린다.
불, 좀비, 바이러스 등 여러 곳에서 동시에 퍼지는 문제에 사용.

```cpp
void fire_bfs() {
    queue<tuple<int,int>> q;
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            if (grid[i][j] == 'F') {
                fire_dist[i][j] = 0;
                q.push({i, j});
            }
    // 이후 일반 BFS...
}
```

### 2. BFS 분리 후 도달 시간 비교

두 주체가 동시에 움직이는 문제에서 한쪽을 먼저 BFS로 전처리하고, 나머지 BFS에서 비교한다.

```cpp
// 지훈이 불보다 엄격히 먼저 도달해야 이동 가능
if (fire_dist[nr][nc] == -1 || jihun_dist[cur] + 1 < fire_dist[nr][nc]) {
    // 이동 가능
}
// 동시 도달(==)은 불가 — 지훈이 탄다
```

### 3. 격자 탈출 감지 패턴

경계 셀을 체크하는 것이 아니라, 격자 밖으로 나가는 순간을 탈출로 처리한다.

```cpp
if (!inRange(next_row, next_col)) {
    return dist[cur_row][cur_col] + 1;
}
```
